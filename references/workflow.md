# 工作流

## 脑内清空

先让用户无序倾倒所有想到的事，不要立刻排序。只在用户停下来后再澄清。

依次确认：

1. 哪些有明确截止期。
2. 哪些最占脑子。
3. 哪些不处理会产生实际后果。
4. 哪些是一串步骤，而不是单步任务。
5. 每件事最小的下一步是什么。

## 每日整理

目标是把当前注意力收敛到 1-3 件事：

1. 读取收集箱和任务总览。
2. 移入新事项。
3. 检查等待与阻塞。
4. 从本周推进中选择今日重点。
5. 确认每个今日重点都有可立刻执行的下一步。

## 周复盘

目标是更新长短线规划：

1. 清理完成项和过期项。
2. 检查每个进行中项目是否还有下一步。
3. 把本周必须推进的任务放入总览。
4. 把等待他人、缺信息、条件不成熟的事项放入等待与阻塞。
5. 把暂时不做但不想丢的事项放入暂缓项目。

## 分类规则

| 情况 | 位置 |
|---|---|
| 今天要推进 | `00_任务总览.md` 的“今日重点” |
| 本周要推进 | `00_任务总览.md` 的“本周推进” |
| 多步骤长期事项 | `02_项目清单.md` |
| 等别人、等信息、等条件 | `03_等待与阻塞.md` |
| 未澄清事项 | `01_收集箱.md` |

## 阻塞处理

| 阻塞类型 | 处理方式 |
|---|---|
| 等别人 | 写清等待对象和下一次检查日期 |
| 等资料 | 写清缺什么资料，以及从哪里拿 |
| 范围太大 | 拆成 15-30 分钟内能开始的动作 |
| 不知道怎么做 | 先列一个查资料、问人或试错动作 |
| 不想做但必须做 | 降低启动成本，只定义第一步 |
| 不确定是否还要做 | 放入暂缓，并设置下次检查日期 |
